# Endale — Claude Context

Campaign wiki for **Endale**, a Daggerheart TTRPG campaign.
Pure HTML/CSS/JS — no build tools, no framework. Works as a local file or on GitHub Pages.

## File structure

```
index.html                        ← app shell (nav + content area)
css/
  main.css                        ← layout, sidebar, nav
  cards.css                       ← card component styles
  mapbuilder.css                  ← map builder styles
js/
  app.js                          ← routing, nav, page registry, catalogue
  cards.js                        ← card/roster/reference renderer
  mapbuilder.js                   ← canvas floor planner
  images.js                       ← portrait image picker (art/ folder)
  art-manifest.js                 ← list of art files (auto-generated by update-art-manifest.py)
data/
  characters/
    player-characters.js          ← Nora, Eweram, Sable
    key-individuals.js            ← Ossian, Fenlow, Jhon, Tooth, Mortimer, Endless One
    lionguard.js                  ← Pipkin, Greymane, Kip, Aldous + guard roster
    applecrumb-faction.js         ← Biggus, Rina, Drew, Nessa + Berry/Clide roster
    ratkin.js                     ← Scratchen, Voss, Netch + skirmisher roster
    fonn-civilians.js             ← Featherhides, Applecrumb family, Fonn townsfolk
  locations/
    fonn.js                       ← Fonn, Tower, Veiled Forest, Crow's Nest, Mansion
  story/
    threads.js                    ← Active/mid-term/slow-burn story threads (Reference section)
  sessions/
    recap.js                      ← Current session quick notes (Reference section)
    log.js                        ← Historical session 1–8 archive (accessible via hash)
  world/
    overview.js                   ← Continents, history, lore, staff, compact, curse
  lore/
    items.js                      ← Items of note (staff, letter, cane, key, etc.)
  rules/
    dm-cheatsheet.js              ← Daggerheart combat reference (Reference section)
```

**IMPORTANT: Data scripts must be loaded AFTER js/app.js in index.html.**
Scripts are executed in order; data files call `Endale.registerPage()` which requires
`Endale` to be defined first.

## Sidebar navigation structure

The sidebar has three fixed sections followed by a scrollable Card Groups area:

```
Title / tagline
──────────────────────────
TOOLS (core-nav, locked)
  Active Manager          ← sessions/board — session corkboard
  Map Builder             ← canvas floor planner
──────────────────────────
REFERENCE (ref-nav, locked)
  DM Cheatsheet           ← rules/dm-cheatsheet
  Session Recap           ← sessions/recap
  Story Threads           ← story/threads
  Full Catalogue          ← aggregated card grid (built-in view)
──────────────────────────
CARD GROUPS (nav-tree, editable)
  Characters group
    Player Characters / Key Individuals / Lionguard / etc.
  World group
    Fonn / World Overview / Items of Note / …
──────────────────────────
+ Group   Edit
```

To add a new page to Card Groups:
1. Create `data/<section>/<name>.js`
2. Add `<script src="...">` in index.html
3. Add `{ id, label }` to the relevant section's `children` array in `NAV_DEFAULT` in `js/app.js`

The Reference section is hardcoded in `REF_NAV`. The Tools section is `CORE_NAV`.

## How pages work

Each data file calls `Endale.registerPage(id, data)`. The id is the URL hash
(e.g. `characters/fonn-civilians` → `index.html#characters/fonn-civilians`).

## Card Types

All `cards`-group entries support a `cardType` field that describes their semantic role.
Use this to guide field choices and future renderer enhancements.

### 1. `character` — Individual character
```js
{ cardType: 'character',
  name, role,                          // e.g. 'Pipkin Pipermane', 'Lionguard Captain'
  rank: ['The Lionguard', 'Rank/title'],
  fields: [
    { label: 'Race/Class',   value: '' },
    { label: 'Motivation',   value: '' },
    { label: 'Relationship', value: '' },
    { label: 'Stat block',   value: '' },  // optional
  ],
  gmNote, quote,
  tags: [{ label, cls }]
}
```

### 2. `roster` — Group of characters (use `roster` group type, not `cards`)
```js
{ type: 'roster', title, role, rank, headerClass,
  sections: [{ heading, columns: [], rows: [{ name, badge, cells: [] }] }]
}
```

### 3. `location` — Place with description and points of interest
```js
{ cardType: 'location',
  name, role,                          // e.g. 'The Crow's Nest', 'Tavern'
  rank: ['Fonn', 'District/region'],
  fields: [
    { label: 'Atmosphere',   value: '' },
    { label: 'NPCs present', value: '' },
    { label: 'POI',          value: '' },  // bullet list of notable features
    { label: 'Secrets',      value: '' },
  ],
  gmNote, quote,
  tags: [{ label, cls }]
}
```

### 4. `lore` — World / faction / continent / story block
Use the `reference` group type for lore-heavy content (story threads, world overview).
For shorter inline lore cards use `cards` with:
```js
{ cardType: 'lore',
  name, role,                          // e.g. 'The Compact', 'Faction: Applecrumb'
  rank: ['World Lore', 'Category'],
  fields: [
    { label: 'Summary',  value: '' },
    { label: 'History',  value: '' },
    { label: 'Factions', value: '' },
  ],
  gmNote, quote,
  tags: [{ label, cls }]
}
```

### 5. `poi` — Individual point of interest (puzzle, encounter, trap, secret)
```js
{ cardType: 'poi',
  name, role,                          // e.g. 'The Locked Vault', 'Trap: Pressure Plate'
  rank: ['Location name', 'POI type'],
  fields: [
    { label: 'Type',      value: '' },  // Puzzle / Encounter / Trap / Secret / Treasure
    { label: 'Setup',     value: '' },
    { label: 'Mechanics', value: '' },
    { label: 'Solution',  value: '' },
  ],
  gmNote, quote,
  tags: [{ label, cls }]
}
```

### 6. `item` — Physical item, weapon, artifact, or mundane object
```js
{ cardType: 'item',
  name, role,                          // e.g. 'The Bone Staff', 'Artifact'
  rank: ['Item type', 'Rarity'],
  fields: [
    { label: 'Type',        value: '' },
    { label: 'Properties',  value: '' },
    { label: 'Effect',      value: '' },
    { label: 'Origin',      value: '' },
    { label: 'Current loc', value: '' },
  ],
  gmNote, quote,
  tags: [{ label, cls }]
}
```

## Group types (rendering)

**cards** — collapsible individual cards (characters, locations, items, poi):
```js
{ type: 'cards', headerClass: 'h-feather', entries: [ /* card objects above */ ] }
```

**roster** — wide table card (groups of NPCs, commoner lists):
```js
{ type: 'roster', title, role, rank: [line1, line2], headerClass,
  sections: [{ heading, columns: [], rows: [
    { name, badge: {label, cls}, cells: [] }
  ]}]
}
```

**reference** — always-visible sections; for cheatsheets, session notes, story threads:
```js
{ type: 'reference', title, role, headerClass,
  sections: [
    { heading, kind: 'table',  columns: [], rows: [['cell', ...], ...] },
    { heading, kind: 'pairs',  rows: [{ label, value }, ...] },
    { heading, kind: 'note',   body: '' }
  ]
}
```
